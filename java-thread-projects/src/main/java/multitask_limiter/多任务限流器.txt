项目场景: 设计一个并发任务管理器
- 使用参数Throttle阀门来限流，控制指定数量的Task并发运行
- 超过限流数量的任务将被放到缓存队列中等待执行
- 任务管理器需要不定时的接受新任务
- 不同Task任务执行时间不同
- 不同任务Priority优先级高的先执行
